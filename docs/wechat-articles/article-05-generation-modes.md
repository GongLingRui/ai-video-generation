前面几篇文章，我反复提到了 T2V、I2V、R2V 这三个词。它们分别代表 AI 视频生成的三种模式。但很多人其实不清楚这三种模式有什么区别，更不知道在什么场景下该用哪一种。

今天这篇文章，就来一次讲清楚。

---

## T2V：Text-to-Video，文生视频

**输入**：一段文字描述（prompt）

**输出**：一段视频

这是最直接、最"AI 魔法"的生成方式。你写一段文字，AI 把它变成会动的画面。

### 适合场景

- **创意探索**：脑子里有一个模糊的画面，想看看 AI 会怎么诠释。
- **氛围测试**：想试试某种色调、某种情绪在视频里的效果。
- **无明确参考**：你没有具体的参考图，也不想被某张图片限制。

### 局限性

- **可控性最低**：AI 对 prompt 的理解有随机性，同样的 prompt 每次生成结果都不同。
- **画面细节不确定**：角色的具体长相、场景的具体布局，都无法精确控制。
- **多镜头一致性差**：每个镜头独立生成时，角色和场景很容易"漂移"。

### 平台里的 T2V

在我的平台中，T2V 通常用于两种情况：

1. **第一个镜头**：整条片子还没有任何参考图，第一个镜头只能用 T2V 从零开始。
2. **风格化实验**：用户想试试不同的 prompt 写法会得到什么样的结果，快速迭代。

**使用技巧**：T2V 的 prompt 质量直接决定输出质量。不要只写"一个女孩在跳舞"，要用八要素系统详细描述主体、构图、光线、动作。prompt 越结构化，结果越可控。

## I2V：Image-to-Video，图生视频

**输入**：一张图片 + 一段文字描述

**输出**：一段视频

这是目前实用价值最高的 AI 视频生成模式。你提供一张起始画面，AI 在这个画面的基础上，让它"动"起来。

### 适合场景

- **基于已有素材创作**：你有一张好看的插画、照片、AI 生成图，想让它动起来。
- **保持画面一致性**：用一张参考图锁定角色/场景的视觉特征，避免 T2V 的随机性。
- **链式生成的核心**：前一个镜头的尾帧作为后一个镜头的首帧，靠的就是 I2V。

### 局限性

- **需要准备参考图**：没有好的参考图，I2V 就无从谈起。
- **运动幅度受限**：AI 需要在保持图片内容的基础上生成运动，如果 prompt 要求"镜头快速旋转 360 度"，而参考图是一个正面人像，结果往往会扭曲。
- **首帧决定上限**：参考图的质量、构图、清晰度，直接影响生成视频的质量。

### 平台里的 I2V

I2V 是平台**最常用**的生成模式，也是链式生成的技术基础。

每个 Config Node（镜头配置节点）都可以设置"首帧图"。你可以：

- 手动上传一张图片作为首帧。
- 使用"链式首帧生成"功能，让 AI 自动为每个镜头生成合适的首帧。
- 前一个镜头生成视频后，自动提取尾帧作为下一个镜头的首帧。

**使用技巧**：I2V 的 prompt 不需要重复描述画面里已经有的东西（比如"一个穿红衣服的女孩"），而应该聚焦于"动作和运动"（比如"女孩缓缓转头看向镜头，风吹起她的头发"）。这样 AI 的注意力会更集中在"怎么动"上，而不是"画面里有什么"。

---

## R2V：Reference-to-Video，首尾帧视频

**输入**：首帧图 + 尾帧图 + 文字描述

**输出**：一段视频

R2V 是最高级的生成模式。它不仅告诉 AI"开头长什么样"，还告诉 AI"结尾应该长什么样"。AI 的任务就是在两张图之间，生成一段合理的过渡视频。

### 适合场景

- **精确控制镜头起止**：你知道这个镜头开始和结束的画面分别是什么，想让 AI 补全中间的过程。
- **复杂运动镜头**：比如镜头从全景推到特写，或者从 A 场景过渡到 B 场景。
- **转场特效**：利用首尾帧的对比，生成有创意的转场效果。

### 局限性

- **需要模型支持**：不是所有 AI 视频模型都支持 R2V。我的平台使用火山引擎的 Seedance 1.0 Pro 模型来支持这个功能。
- **首尾帧差异不能太大**：如果首帧是一个人在室内，尾帧是一个人在月球表面，AI 很难生成合理的过渡。
- **生成时间更长**：因为 AI 需要同时满足两张图的约束，计算量更大。

### 平台里的 R2V

在平台的 Config Node 中，你可以同时设置"首帧图"和"尾帧图"。当两者都存在时，系统会自动切换到 Seedance 1.0 Pro 模型，使用 R2V 模式生成。

一个典型的 R2V 使用场景：

> 镜头 1：全景，主角站在城市天台，背对镜头。→ 镜头推近，最终定格在主角侧脸的特写。

首帧是天台全景，尾帧是侧脸特写。R2V 会自动生成一个"推轨镜头"的过渡过程。

**使用技巧**：R2V 的两张图在色调、风格上要尽量一致。如果首帧是暖色调、尾帧是冷色调，AI 可能会在中间插入不自然的颜色突变。最好在生成首尾帧时，用同样的 prompt 和参数。

## 三种模式怎么选？一张表告诉你

| 场景 | 推荐模式 | 理由 |
|------|---------|------|
| 第一个镜头，没有任何参考 | T2V | 从零开始，探索可能性 |
| 有参考图，想让画面动起来 | I2V | 锁定画面特征，保证一致性 |
| 链式生成中的常规镜头 | I2V | 用前一个镜头的尾帧作为首帧 |
| 需要精确控制镜头起止 | R2V | 首尾帧双重约束，运动更可控 |
| 想测试不同 prompt 的效果 | T2V | 快速迭代，无需准备素材 |
| 复杂转场或运动镜头 | R2V | AI 补全中间过程，创意空间大 |
| 批量生成，追求一致性 | I2V + 链式 | 首尾帧传递，保证全片统一 |

## 平台的智能模式切换

一个贴心的设计是：用户**不需要手动选择模式**。

平台会根据你的配置，自动判断用哪种模式：

- 只有 prompt → T2V
- 有 prompt + 首帧图 → I2V
- 有 prompt + 首帧图 + 尾帧图 → R2V

你只需要关注"我要这个镜头长什么样"，平台会帮你选择最合适的技术路径。
