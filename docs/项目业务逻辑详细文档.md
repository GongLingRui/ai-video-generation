# AI Video Generation Platform - 业务逻辑详细文档

## 一、项目概述

这是一个基于 Next.js + React Flow 的 AI 分镜视频生成平台。用户可以通过自然语言描述，利用 AI 自动生成分镜剧本（Storyboard），然后逐个镜头生成视频片段，并通过首尾帧衔接实现多段视频的视觉连贯性。

### 1.1 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | Next.js 15 (App Router) + React |
| 状态管理 | Zustand |
| 可视化编辑器 | React Flow |
| AI 集成 | Vercel AI SDK + Volcano Engine ARK API |
| 样式 | Tailwind CSS + shadcn/ui |
| 后端 | Next.js API Routes |
| 持久化 | Supabase |

### 1.2 核心 API 服务

- **视频生成**: Seedance (火山方舟 ARK API)
- **图片生成**: Seedream (火山方舟 ARK API)
- **AI 对话/Storyboard 生成**: Doubao (Volcano Engine)

## 二、数据模型

### 2.1 ConfigNodeData（配置节点）

每个镜头（Shot）对应一个配置节点，是视频生成的核心数据结构。

```typescript
interface ConfigNodeData {
  // ===== 基础标识 =====
  shotId?: number;           // 镜头编号 (1, 2, 3...)，由添加顺序决定
  title?: string;            // 镜头标题（可选）

  // ===== 核心内容 =====
  prompt: string;            // 主提示词/描述
  negativePrompt?: string;   // 负面提示词（需要避免的内容）
  techniques: Technique[];   // 应用的摄影技术（多选）

  // ===== 视频规格 =====
  ratio: VideoRatio;         // 宽高比: '16:9' | '9:16' | '1:1' | '4:3' | '3:4' | '21:9'
  duration: VideoDuration;    // 时长: 4 | 5 | 8 | 12 (秒)
  resolution: VideoResolution; // 分辨率: '480p' | '720p' | '1080p'
  status: NodeStatus;        // 状态: 'idle' | 'generating' | 'done' | 'error'

  // ===== 首尾帧图片（首尾帧参考模式） =====
  firstFrameUrl?: string;    // 首帧图片 URL（用户上传或 AI 生成）
  lastFrameUrl?: string;     // 尾帧图片 URL（用户上传或 AI 生成）

  // ===== 参考图（用于图生视频时的风格/角色参考） =====
  referenceImageUrls?: string[];

  // ===== 参考帧模式（覆盖全局设置） =====
  referenceFrameMode?: 'first-last' | 'multi-frame';

  // ===== AI 图片生成状态 =====
  imageGenStatus: 'idle' | 'generating' | 'selecting' | 'done';
  currentImageGrid?: ImageGrid;   // 当前生成的九宫格
  imageGenHistory: ImageGrid[];  // 历史生成记录
  availableGenModes: ImageGenMode[]; // 该节点可用的生成模式

  // ===== 三阶段流程字段（由 AI 自动填充） =====
  sceneType?: 'opening' | 'dialogue' | 'action' | 'climax' | 'ending'; // 场景类型
  shotSize?: 'extreme-close-up' | 'close-up' | 'medium' | 'wide' | 'extreme-wide'; // 景别
  transition?: string;         // 转场方式
  coreInfo?: string;            // 核心信息（一句话描述）
  visualKeywords?: string[];     // 视觉关键词
  characterFeatures?: string;   // 角色特征描述
  sceneFeatures?: string;       // 场景特征描述
  consistencyNotes?: string;    // 一致性要求

  // ===== AI 精选提示词 =====
  selectedPrompt?: string;           // AI 从 alternativePrompts 中精选的最佳提示词
  alternativePrompts?: string[];      // AI 生成的多个备选提示词

  // ===== 首尾帧提示词（用于 AI 生成首尾帧图片） =====
  firstFramePrompt?: string;   // 首帧图片生成提示词
  lastFramePrompt?: string;     // 尾帧图片生成提示词

  // ===== 九宫格模式 =====
  multiFramePrompt?: string;    // 九宫格生成的主提示词
  multiFrameImageUrl?: string;  // 九宫格生成的参考图 URL

  // ===== 八要素（详细视觉描述） =====
  composition?: string;         // 构图：主体位置、视角、景深
  lighting?: string;           // 光线：光源方向、光质、对比度
  subject?: string;             // 主体：人物/物体特征、状态
  background?: string;         // 背景：环境、空间、色彩
  actionMovement?: string;      // 动作/运动：主体动作、镜头运动
  textOverlay?: string;        // 文字叠加：字幕内容（'无' 表示无文字）
  transitionDetail?: string;   // 转场细节：类型、时机、效果
  audio?: string;              // 音频：音乐、音效、旁白（'无旁白' 表示无配音）
}
```

### 3.1 CanvasStore（画布状态）

核心状态仓库，管理所有节点、边、全局配置和三阶段数据。

**请求体：**
```typescript
{
  // 基础字段
  prompt: string;
  negativePrompt?: string;
  techniques: Technique[];
  ratio: VideoRatio;
  duration: number;
  resolution: VideoResolution;
  videoModelKey?: VideoModelKey;

  // 首尾帧
  firstFrameUrl?: string;
  lastFrameUrl?: string;

  // 参考图
  referenceImageUrls?: string[];

  // 增强元数据
  sceneType?: string;
  shotSize?: string;
  transition?: string;
  coreInfo?: string;
  visualKeywords?: string[];
  characterFeatures?: string;
  sceneFeatures?: string;
  consistencyNotes?: string;
  selectedPrompt?: string;
  alternativePrompts?: string[];
  firstFramePrompt?: string;
  lastFramePrompt?: string;

  // 八要素
  composition?: string;
  lighting?: string;
  subject?: string;
  background?: string;
  actionMovement?: string;
  textOverlay?: string;
  transitionDetail?: string;
  audio?: string;
}
```

**响应：**
```typescript
{ taskId: string; prompt: string; }
```

**业务逻辑：**
1. 验证首帧/尾帧 URL 格式（必须是 HTTPS）
2. 验证参考图数量（最多 2 张）
3. 如果有增强元数据，使用 `buildEnhancedPrompt()` 构建完整提示词；否则使用 `buildPrompt()`
4. 调用 `createVideoTask()` 创建任务
5. **自动模型切换**：如果同时提供了 `firstFrameUrl` 和 `lastFrameUrl`（r2v 模式），自动从 `SEEDANCE_MODEL` 切换到 `SEEDANCE_LITE_MODEL`

#### GET `/api/generate/[taskId]` - 查询视频任务状态

**响应：**
```typescript
{
  status: 'queued' | 'running' | 'succeeded' | 'failed' | 'expired';
  videoUrl?: string;
  lastFrameUrl?: string;
  error?: string;
}
```

## 五、核心业务逻辑

### 5.1 视频生成流程

```
用户点击"生成视频"
        ↓
useVideoGeneration.generate(nodeData)
        ↓
┌─ 检查参考帧模式 ─────────────────────┐
│  multi-frame: 使用 multiFrameImageUrl │
│  first-last: 使用 firstFrameUrl +     │
│              lastFrameUrl             │
└──────────────────────────────────────┘
        ↓
获取链式衔接的首帧
（getPreviousNodeLastFrame）
        ↓
POST /api/generate
        ↓
┌─ 自动模型切换 ───────────────────────┐
│  如果 firstFrame + lastFrame 同时存在 │
│  且 model === SEEDANCE_MODEL         │
│  → 自动切换到 SEEDANCE_LITE_MODEL    │
│  （1.5 Pro 不支持 r2v 模式）          │
└──────────────────────────────────────┘
        ↓
Seedance API 返回 taskId
        ↓
轮询 GET /api/generate/[taskId]
（3秒间隔，最多60次）
        ↓
┌─ 成功 ────────────────────────────────┐
│  setVideoResult(nodeId, videoUrl,     │
│                 prompt, lastFrameUrl) │
│  updateNodeStatus(nodeId, 'done')    │
└──────────────────────────────────────┘
        ↓
┌─ 失败/超时 ───────────────────────────┐
│  toast.error(message)                │
│  updateNodeStatus(nodeId, 'error')   │
└──────────────────────────────────────┘
```

### 5.2 链式生成流程（Chain Generation）

链式生成是多段视频连续生成的核心功能，通过尾帧→首帧的自动衔接，实现视觉连贯性。

```
shots = [shot1, shot2, shot3, ...]
previousLastFrameUrl = null

For i = 0 to shots.length - 1:
        ↓
  ┌─ 判断首帧来源 ─────────────────────────┐
  │  if i === 0 OR previousLastFrameUrl === null: │
  │    → AI 生成首帧（firstFramePrompt）     │
  │  else:                                    │
  │    → 使用上一镜尾帧（previousLastFrameUrl）│
  └─────────────────────────────────────────┘
        ↓
  AI 生成尾帧（lastFramePrompt）
        ↓
  更新节点 i：
    - firstFrameUrl
    - lastFrameUrl
        ↓
  previousLastFrameUrl = 尾帧 URL
        ↓
  2秒延迟（避免 API 限流）
        ↓
  继续下一镜
```

**关键特性：**
- 顺序执行（非并行），避免 API 限流
- 支持中断（AbortController）
- 部分失败继续（reportProgress）
- 每个镜头的首帧来源由前一个镜头的尾帧决定

### 5.3 参考帧模式

系统在两个层级管理参考帧模式：

**全局级别**（GlobalConfig.referenceFrameMode）：
- `first-last`：首尾帧参考模式
- `multi-frame`：九宫格多帧参考模式

**节点级别**（ConfigNodeData.referenceFrameMode）：
- 可选覆盖全局设置

#### 5.3.1 首尾帧参考模式（first-last）

首帧和尾帧图片共同约束视频的起止画面，AI 生成两帧之间的过渡动画。

```
content: [
  { type: 'image_url', image_url: { url: firstFrameUrl }, role: 'first_frame' },
  { type: 'image_url', image_url: { url: lastFrameUrl }, role: 'last_frame' },
  { type: 'text', text: prompt }
]
```

**模型限制：**
- `doubao-seedance-1-5-pro-251215`：不支持 r2v 模式（会报错）
- `doubao-seedance-1-0-pro-250528`：支持 r2v 模式

#### 5.3.2 九宫格多帧参考模式（multi-frame）

使用九宫格参考图作为视频生成的参考，不控制具体首尾帧。

```
content: [
  { type: 'image_url', image_url: { url: multiFrameImageUrl }, role: 'reference_image' },
  { type: 'text', text: prompt }
]
```

**限制：**
- 不支持 1080p 分辨率
- 不支持 ratio 为 `adaptive`

### 5.5 提示词构建

#### 5.5.1 基础提示词构建（buildPrompt）

适用于没有增强元数据的简单场景。

```typescript
buildPrompt(userPrompt, techniques, negativePrompt)

```

#### 5.5.2 增强提示词构建（buildEnhancedPrompt）

适用于有完整八要素和三阶段数据的场景，生成更详细的视频描述。

```typescript
buildEnhancedPrompt(nodeData)

// "核心提示词。
//  场景类型: 开场镜头，景别: 特写，转场: 淡入淡出。

//  【角色特征】...
//  【场景特征】...
//  【一致性要求】...
//  镜头: 推近, 斯坦尼康。
//  避免: ..."
```

### 5.6 八要素系统

八要素是 AI 分镜的核心输出规范，确保每个镜头都有完整的视觉描述：

| 要素 | 说明 | 示例 |
|------|------|------|
| **subject** | 主体：人物/物体特征 | "Q版萌系乌龟举着小奖杯" |
| **composition** | 构图：主体位置、视角、景深 | "冲线的乌龟位于画面居中位置，平角度居中构图，深景深" |
| **lighting** | 光线：光源方向、光质、氛围 | "顶光漫射柔光，暖调明亮氛围" |
| **background** | 背景：环境、色彩 | "终点线挂着彩色小旗子，森林小动物欢呼" |
| **actionMovement** | 动作/运动：主体动作、镜头运动 | "乌龟缓慢冲过终点线，镜头固定不动" |
| **textOverlay** | 文字叠加：字幕内容 | "无" 或 "第1秒：加油！" |
| **transitionDetail** | 转场细节：类型、时机 | "fade淡出，乌龟举奖杯庆祝时慢慢淡出" |
| **audio** | 音频：音乐、音效、旁白 | "背景音乐到达欢快高潮段，搭配欢呼声" |

## 七、Hooks 详解

### 7.1 useVideoGeneration

管理单个节点的视频生成生命周期。

```typescript
const { status, generate, cancel } = useVideoGeneration(nodeId)
```

**generate(nodeData: ConfigNodeData)：**
1. 创建 AbortController
2. 更新节点状态为 generating
3. 获取全局配置和链式首帧
4. 根据参考帧模式确定图片参数
5. POST /api/generate
6. 轮询任务状态（3秒间隔，最多60次）
7. 更新 store：setVideoResult
8. 处理错误和中断

### 7.2 useImageGeneration

管理图片生成（九宫格模式）。

```typescript
const { status, generate } = useImageGeneration(nodeId)
```

**generate(mode: ImageGenMode, prompt: string)：**
1. POST /api/generate-image
2. 如果返回 imageUrl：直接使用
3. 如果返回 taskId：轮询直到完成
4. 更新 store

### 7.3 useChainGeneration

管理多镜头链式生成的完整流程。

```typescript
const {
  generateShotFrames,      // 生成单个镜头的首尾帧
  startChainGeneration,    // 开始完整链式生成
  abortChainGeneration,    // 中断生成
  regenerateShotFrames     // 重新生成单个镜头
} = useChainGeneration()
```

**startChainGeneration(shots, configNodeIds, ...) 流程：**

```
输入: shots[], configNodeIds[]

previousLastFrameUrl = null
results = []

for i = 0 to shots.length:
  shot = shots[i]
  nodeId = configNodeIds[i]

  // 确定首帧来源
  if i === 0 or not previousLastFrameUrl:
    firstFrameUrl = null (将 AI 生成)
    firstFrameUrl = previousLastFrameUrl (继承上一镜尾帧)

  // 生成尾帧
  lastFrameUrl = await generateShotFrames(
    prompt,
    'last-frame',
    lastFramePrompt,
    signal
  )

  // 更新节点
  updateNodeFirstFrame(nodeId, firstFrameUrl)
  updateNodeLastFrame(nodeId, lastFrameUrl)

  // 2 秒延迟
  await sleep(2000)

  previousLastFrameUrl = lastFrameUrl

// 返回结果
return { successCount, failedCount }
```

## 十一、关键业务规则

1. **节点 ID 命名**：`config-{counter}` 和 `video-{counter}`，counter 共享
2. **删除节点**：同时删除对应的 ConfigNode 和 VideoNode
3. **链式衔接**：每个视频节点的尾帧自动成为下一节点的首帧来源
4. **版本历史**：VideoNode 保存所有生成版本，支持切换
5. **技术应用**：可同时应用多个技术，系统合并到提示词中
6. **九宫格限制**：不支持 1080p 和 adaptive ratio
7. **首尾帧 URL 验证**：必须使用 HTTPS
8. **参考图限制**：最多 2 张参考图
9. **生成超时**：60 次轮询（约 3 分钟）
10. **链式生成延迟**：每镜之间 2 秒延迟避免 API 限流
