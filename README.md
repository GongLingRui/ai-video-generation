## 😫 Do These Pain Points Sound Familiar?

- **You have a vision but can't write it down** — You want to shoot a short video, but writing the storyboard takes an entire day, and bringing your idea to life feels endlessly far away.
- **AI-generated videos look like Frankensteins** — You generate clips with various tools, but the styles and actions don't connect, creating a jarring mess when edited together.
- **Clients/bosses suddenly change requirements** — Scenes get scrapped and rebuilt from scratch, forcing you to rework storyboards, scripts, and reference images, making all-nighters the norm.
- **You don't know camera terminology, so you can't control shots precisely** — You want a "push-in + slow motion + cinematic look," but have no idea how to express that in prompts.
- **Multiple video clips can't maintain character/scene consistency** — The protagonist looks one way in this second, and has a completely different face in the next.

**If any of the above resonates with you, this platform was built for you.**

## 💡 In a Nutshell

**AI Video Storyboard Platform** is a one-stop workstation from **Creative Dialogue → AI Storyboard Planning → Fine Control → Batch Video Generation**. Just describe your ideas in natural language, and AI automatically breaks down scenes, writes storyboards, assigns camera techniques, and generates **stylistically consistent** multi-segment videos.

## 🎯 Features

### 1. AI Storyboard Generation (Core Feature)

Create video storyboards through natural language dialogue with AI:

```
User: "I want to make an animated short about a turtle competing in a sports meet, in an anime style"
```

AI automatically completes a three-stage generation:

#### Stage 1: Concept Phase

```
┌─────────────────────────────────────────────────────────────┐
│  Creative Analysis                                           │
│  - Core Concept: Turtle's inspirational story                │
│  - Style Preset: Anime (Japanese animation)                  │
│  - Target Audience: All ages                                 │
│  - Target Duration: 30 seconds                               │
├─────────────────────────────────────────────────────────────┤
│  Scene Breakdown                                             │
│  ┌────────┬────────┬────────┬────────┐                     │
│  │ Scene 1│ Scene 2│ Scene 3│ Scene 4│  (4 scenes total)   │
│  │ Setup  │Conflict│ Climax │Ending  │                     │
│  └────────┴────────┴────────┴────────┘                     │
└─────────────────────────────────────────────────────────────┘
```

#### Stage 2: Planning Phase

```
┌─────────────────────────────────────────────────────────────┐
│  Storyboard Planning                                         │
│                                                              │
│  Total Shots: 6  │  Total Duration: 28s                      │
│                                                              │
│  ┌────┬────┬────┬────┬────┬────┐                           │
│  │Shot1│Shot2│Shot3│Shot4│Shot5│Shot6│                      │
│  │ 5s │ 4s │ 6s │ 5s │ 4s │ 4s │                           │
│  │Push│CU  │MS  │WS  │Slow│Slow│                           │
│  │In  │    │    │    │Mo  │Mo  │                           │
│  └────┴────┴────┴────┴────┴────┘                           │
└─────────────────────────────────────────────────────────────┘
```

#### Stage 3: Storyboard Phase

Each shot contains a complete **Eight-Element** description:

| Element | Example Output |
|---------|---------------|
| **Subject** | Chibi cute turtle holding a small trophy |
| **Composition** | Turtle crossing the finish line centered, eye-level centered composition, deep depth of field |
| **Lighting** | Top diffused soft light, warm and bright atmosphere |
| **Background** | Finish line with colorful small flags, forest animals cheering |
| **Action/Movement** | Turtle slowly crosses the finish line, camera stays static |
| **Text Overlay** | None |
| **Transition Detail** | Fade out, slowly fading as turtle raises trophy in celebration |
| **Audio** | Background music reaches cheerful climax, paired with cheering sounds |

### 4. Three Video Generation Modes

#### T2V (Text-to-Video)

```
Input: "A turtle running in the forest"
   ↓
Output: [Generated video]
```

#### I2V (Image-to-Video)

```
First frame image + Text description
   ↓
Output: [Video generated from the first frame]
```

#### R2V (Reference-to-Video) (Requires Seedance 1.0 Pro)

```
First frame image + Last frame image + Text description
   ↓
Output: [Video transitioning from first to last frame]
```

## 🚀 Quick Start

### Requirements

- Node.js 18+
- npm / yarn / pnpm
- Supabase account
- Volcano Engine ARK API Key

### Installation

```bash
# 1. Clone the project
git clone https://github.com/your-username/ai-video.git
cd ai-video

# 2. Install dependencies
npm install

# 3. Configure environment variables
cp .env.example .env.local
```

### Environment Variables

```env
# Supabase Config
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key

# Volcano Engine ARK API Config
ARK_API_KEY=your-volcengine-api-key
ARK_ENDPOINT_ID=your-doubao-endpoint-id

# Optional: Model overrides (usually not needed)
SEEDANCE_MODEL=doubao-seedance-1-5-pro-251215
SEEDANCE_LITE_MODEL=doubao-seedance-1-0-pro-250528
IMAGE_MODEL=doubao-seedream-5-0-260128
```

### Start Development Server

```bash
npm run dev
```

Visit http://localhost:3000 to use.

### Supabase Deployment Config

#### 1. Create Supabase Project

1. Visit [supabase.com](https://supabase.com) to create a new project
2. Wait for project creation to complete, then get the project URL and Anon Key

#### 2. Execute Database Schema

Run [`docs/supabase/schema.sql`](docs/supabase/schema.sql) in the Supabase SQL Editor:

```sql
-- Or import the SQL file directly in the Supabase Dashboard
```

#### 3. Configure Row Level Security (RLS)

All tables have RLS policies configured to ensure users can only access their own data:

```sql
-- Enable RLS (usually enabled by default)
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE projects ENABLE ROW LEVEL SECURITY;
ALTER TABLE assets ENABLE ROW LEVEL SECURITY;
ALTER TABLE generation_tasks ENABLE ROW LEVEL SECURITY;
```

#### 4. Configure Storage Bucket

1. Create a `project-media` storage bucket in the Supabase Dashboard
2. Set it to **Public** bucket
3. Configure storage policies to allow uploads and access

#### 5. Configure Auth Providers (Optional)

In Supabase Dashboard → Authentication → Providers, configure:

- Email/Password login (enabled by default)
- Google OAuth (optional)
- GitHub OAuth (optional)

### Volcano Engine ARK API Configuration

#### 1. Get API Key

1. Visit [Volcano Engine ARK Console](https://www.volcengine.com/product/ark)
2. Create an API Key
3. Get the Endpoint ID (Doubao model)

#### 2. Model Reference

| Model | Purpose | Description |

## 🔌 API Reference

#### POST `/api/generate`
Create a video generation task

```typescript
// Request
{
  prompt: string;              // Prompt
  negativePrompt?: string;      // Negative prompt
  techniques?: Technique[];    // Camera techniques
  ratio: VideoRatio;           // Aspect ratio
  duration: VideoDuration;     // Duration
  firstFrameUrl?: string;      // First frame image
  lastFrameUrl?: string;       // Last frame image
  // ... eight-element fields
}

// Response
{ taskId: string; }
```

Query video task status

```typescript
// Response
{
  status: 'queued' | 'running' | 'succeeded' | 'failed';
  videoUrl?: string;
  lastFrameUrl?: string;
}
```

## 🎨 The Eight-Element System

The Eight-Element system is the core output specification for AI storyboarding, ensuring every shot has a complete, professional visual description:

| Element | English | Keyword | Description | Example |
|---------|---------|---------|-------------|---------|
| **subject** | Subject | Subject | Character/object features | "Chibi cute turtle holding a small trophy" |
| **composition** | Composition | Composition | Subject position, perspective, depth of field | "Eye-level centered composition, deep depth of field" |
| **lighting** | Lighting | Lighting | Light source direction, quality | "Top diffused soft light, warm and bright" |
| **background** | Background | Background | Environment, space, color | "Finish line with colorful small flags" |
| **actionMovement** | Action/Movement | Action/Movement | Subject action, camera movement | "Turtle slowly crosses the finish line" |
| **textOverlay** | Text Overlay | Text Overlay | Subtitle content | "None" or "At 1s: Go for it!" |
| **transitionDetail** | Transition | Transition | Type, timing, effect | "Fade out, slowly fading" |
| **audio** | Audio | Audio | Music, sound effects, narration | "Cheerful background music + cheering" |

### Eight-Element Prompt Template

```
[Visual Elements]
Subject: {subject}
Composition: {composition}
Lighting: {lighting}
Background: {background}
Action: {actionMovement}
Transition Detail: {transitionDetail}
Audio: {audio}

[Consistency Requirements]
{consistency_notes}
```

## 🧩 Component Reference

### ConfigNode

Each shot on the canvas corresponds to a config node, containing:

- **Prompt Editor**: Editable prompt text box
- **Technique Tags**: Applied camera techniques, clickable to remove
- **Eight-Element Editor**: Collapsible detailed visual description
- **Video Specs**: ratio, duration, resolution selectors
- **First/Last Frame Panel**: Reference frame mode toggle and image management
- **Generate Button**: Trigger video generation

### VideoNode

One-to-one mapping with ConfigNode, displaying the generated video:

- **Video Player**: HTML5 video with control bar
- **Last Frame Preview**: Thumbnail of the video's final frame
- **Status Display**: idle / generating / done / error
- **Action Bar**: Regenerate, download video
- **Version History**: Support switching to historical versions

### LeftPanel (Left Technique Library)

- **Category Accordions**: Angle / View / Movement / Style
- **Technique Chips**: Click to apply to selected node
- **Hover Tooltip**: Display technique description and keywords

### RightPanel (Right AI Chat)

- **GlobalConfigBar**: Global model and specs settings
- **ChatMessages**: Chat history
- **StoryboardTabs**: Storyboard result display
  - **OverviewTab**: Overview table
  - **DetailsTab**: Detailed cards

## 🔐 Security Features

- **Row Level Security (RLS)**: All tables have row-level security policies enabled
- **User Data Isolation**: Users can only access their own data
- **API Key Hidden**: Sensitive configs exist only on the server side
- **HTTPS Enforced**: All external resource URLs must be HTTPS
