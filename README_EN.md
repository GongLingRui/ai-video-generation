## 😫 Do these pain points sound familiar?

- **You have a vision in your head but can't write it down** — you want to make a short video, but writing a storyboard script takes a whole day, making the gap between idea and execution feel endless.
- **AI-generated videos look like a "frankenstein" edit** — you generate clips using various tools, but the styles and actions don't connect, resulting in a jarring final product.
- **Clients / bosses change requirements on a whim** — you have to scrap everything and start over: storyboards, scripts, reference images... pulling all-nighters becomes the norm.

- **Multi-clip videos can't maintain character / scene consistency** — the main character looks like this in one second, and has a completely different face in the next.

**If any of the above resonates with you, this platform was built for you.**

## 🎯 Features

### 1. AI Storyboard Generation (Core Feature)

Create video storyboards through natural language conversation with AI:

```
User: "I want to make an animated short about a turtle participating in a sports meet, in anime style"
```

AI automatically completes a three-stage generation:

#### Stage 1: Concept

```
┌─────────────────────────────────────────────────────────────┐
│  Creative Analysis                                           │
│  - Core Idea: Turtle's inspirational story                  │
│  - Style Preset: Anime                                       │
│  - Target Audience: All ages                                │
│  - Target Duration: 30s                                     │
├─────────────────────────────────────────────────────────────┤
│  Scene Breakdown                                             │
│  ┌────────┬────────┬────────┬────────┐                     │
│  │ Scene 1│ Scene 2│ Scene 3│ Scene 4│  (4 scenes total)   │
│  │ Setup  │Conflict│Climax  │Ending  │                     │
│  └────────┴────────┴────────┴────────┘                     │
└─────────────────────────────────────────────────────────────┘
```

#### Stage 2: Planning

```
┌─────────────────────────────────────────────────────────────┐
│  Storyboard Planning                                         │
│                                                              │
│  Total Shots: 6  │  Total Duration: 28s                      │
│                                                              │
│  ┌────┬────┬────┬────┬────┬────┐                           │
│  │ S1 │ S2 │ S3 │ S4 │ S5 │ S6 │                           │
│  │ 5s │ 4s │ 6s │ 5s │ 4s │ 4s │                           │
│  │Push│ CU │ MS │ WS │Slo-│Slo-│                           │
│  │ In │    │    │    │mo  │mo  │                           │
│  └────┴────┴────┴────┴────┴────┘                           │
└─────────────────────────────────────────────────────────────┘
```

#### Stage 3: Storyboard

Each shot contains a complete **8-element** description:

| Element | Example Output |
|---------|---------------|
| **Subject** | Chibi cute turtle holding a small trophy |
| **Composition** | Turtle crossing finish line centered, eye-level centered composition, deep depth of field |
| **Lighting** | Top diffused soft light, warm and bright atmosphere |
| **Background** | Finish line with colorful small flags, forest animals cheering |
| **Action / Movement** | Turtle slowly crosses finish line, camera is static |
| **Text Overlay** | None |
| **Transition Detail** | Fade out, slowly fading as turtle lifts trophy to celebrate |
| **Audio** | Background music reaches cheerful climax, paired with cheering sounds |

### 4. Three Video Generation Modes

#### T2V (Text-to-Video)

```
Input: "A turtle running in the forest"
   ↓
Output: [Generated video]
```

#### I2V (Image-to-Video)

```
First frame image + text description
   ↓
Output: [Video starting from the first frame]
```

#### R2V (Reference-to-Video) (requires Seedance 1.0 Pro)

```
First frame image + last frame image + text description
   ↓
Output: [Video transitioning from first frame to last frame]
```

## 🚀 Quick Start

### Prerequisites

- Node.js 18+
- npm / yarn / pnpm
- Supabase account
- Volcano Engine ARK API Key

### Installation

```bash
# 1. Clone the project
git clone https://github.com/your-username/ai-video.git
cd ai-video

# 2. Install dependencies
npm install

# 3. Configure environment variables
cp .env.example .env.local
```

### Environment Variables

```env
# Supabase Configuration
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key

# Volcano Engine ARK API Configuration
ARK_API_KEY=your-volcengine-api-key
ARK_ENDPOINT_ID=your-doubao-endpoint-id

# Optional: Model overrides (usually no need to change)
SEEDANCE_MODEL=doubao-seedance-1-5-pro-251215
SEEDANCE_LITE_MODEL=doubao-seedance-1-0-pro-250528
IMAGE_MODEL=doubao-seedream-5-0-260128
```

### Start Development Server

```bash
npm run dev
```

Visit http://localhost:3000 to use the app.

### Supabase Deployment Configuration

#### 1. Create a Supabase Project

1. Visit [supabase.com](https://supabase.com) to create a new project
2. Wait for project creation to complete, then get your project URL and Anon Key

#### 2. Execute Database Schema

Run [`docs/supabase/schema.sql`](docs/supabase/schema.sql) in the Supabase SQL Editor:

```sql
-- Or import the SQL file directly in the Supabase Dashboard
```

#### 3. Configure Row Level Security (RLS)

All tables have RLS policies configured to ensure users can only access their own data:

```sql
-- Enable RLS (usually enabled by default)
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE projects ENABLE ROW LEVEL SECURITY;
ALTER TABLE assets ENABLE ROW LEVEL SECURITY;
ALTER TABLE generation_tasks ENABLE ROW LEVEL SECURITY;
```

#### 4. Configure Storage Bucket

1. Create a `project-media` bucket in the Supabase Dashboard
2. Set it as a **Public** bucket
3. Configure storage policies to allow uploads and access

#### 5. Configure Authentication Providers (Optional)

In Supabase Dashboard → Authentication → Providers, configure:

- Email/Password login (enabled by default)
- Google OAuth (optional)
- GitHub OAuth (optional)

#### 1. Get API Key

1. Visit the [Volcano Engine ARK Console](https://www.volcengine.com/product/ark)
2. Create an API Key
3. Get the Endpoint ID (Doubao model)

## 🔌 API Reference

### Video Generation

#### POST `/api/generate`
Create a video generation task

```typescript
// Request
{
  prompt: string;              // Prompt
  negativePrompt?: string;      // Negative prompt
  techniques?: Technique[];    // Cinematography techniques
  ratio: VideoRatio;           // Aspect ratio
  duration: VideoDuration;     // Duration
  firstFrameUrl?: string;      // First frame image
  lastFrameUrl?: string;       // Last frame image
  // ... 8-element fields
}

// Response
{ taskId: string; }
```

#### GET `/api/generate/[taskId]`
Query video task status

```typescript
// Response
{
  status: 'queued' | 'running' | 'succeeded' | 'failed';
  videoUrl?: string;
  lastFrameUrl?: string;
}
```

### Image Generation

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/generate-image` | POST | Create image generation task |
| `/api/generate-image/[taskId]` | GET | Query image task status |
| `/api/generate-frame` | POST | Generate first/last frame images |
| `/api/generate-multi-frame` | POST | Generate nine-grid images |

### AI Chat

#### POST `/api/chat`
AI storyboard chat, supports three tools:

- `generateConcept` - Generate concept
- `generatePlanning` - Generate planning
- `generateStoryboard` - Generate storyboard

## 🎨 8-Element System Explained

The 8-element system is the core output specification for AI storyboarding, ensuring each shot has a complete, professional visual description:

| Element | Chinese Name | English Keyword | Description | Example |
|---------|-------------|-----------------|-------------|---------|
| **subject** | Subject | Subject | Character / object features | "Chibi cute turtle holding a small trophy" |
| **composition** | Composition | Composition | Subject position, angle, depth of field | "Eye-level centered composition, deep depth of field" |
| **lighting** | Lighting | Lighting | Light source direction, quality | "Top diffused soft light, warm and bright" |
| **background** | Background | Background | Environment, space, color | "Finish line with colorful small flags" |
| **actionMovement** | Action / Movement | Action/Movement | Subject action, camera movement | "Turtle slowly crosses finish line" |
| **textOverlay** | Text Overlay | Text Overlay | Subtitle content | "None" or "At 1s: Go for it!" |
| **transitionDetail** | Transition Detail | Transition | Type, timing, effect | "Fade out, slowly fading" |
| **audio** | Audio | Audio | Music, sound effects, narration | "Cheerful BGM + cheering sounds" |

### 8-Element Prompt Template

```
【Visual Elements】
Subject: {subject}
Composition: {composition}
Lighting: {lighting}
Background: {background}
Action: {actionMovement}
Transition Detail: {transitionDetail}
Audio: {audio}

【Consistency Requirements】
{consistency_notes}
```

## 🎮 User Guide

### Basic Operations

1. **Create New Project**: Click the "Create Project" button on the homepage
2. **AI Storyboard**: Enter a creative description in the right panel and chat with AI to generate a storyboard
3. **Add to Canvas**: Click the "Add to Canvas" button on the storyboard card
4. **Adjust Techniques**: Select a node and click a tag in the left technique library to apply
5. **Generate Video**: Click the "Generate" button on the node
6. **Chain Generate**: Click the "Chain Generate All" button at the bottom

### Node Operations

| Operation | Method |
|-----------|--------|
| Add Node | Double-click empty canvas area |
| Delete Node | Select node → Delete/Backspace |
| Select Node | Click the node |
| Edit Prompt | Double-click node or click edit icon |
| Apply Technique | Click a tag in the left technique library |

### Shortcuts

| Shortcut | Function |
|----------|----------|
| `Delete` / `Backspace` | Delete selected node |
| `Double-click canvas` | Add empty config node |
| `Ctrl/Cmd + S` | Save project |

## 🧩 Component Reference

### ConfigNode

Each shot on the canvas corresponds to a config node, containing:

- **Prompt Editor**: Editable prompt text box
- **Technique Tags**: Applied cinematography techniques, clickable to remove
- **8-Element Editor**: Collapsible detailed visual description
- **Video Specs**: ratio, duration, resolution selectors
- **First/Last Frame Panel**: Reference frame mode switch and image management
- **Generate Button**: Triggers video generation

### VideoNode

Corresponds one-to-one with ConfigNode, displaying the generated video:

- **Video Player**: HTML5 video with controls
- **Last Frame Preview**: Thumbnail of the video's last frame
- **Status Display**: idle / generating / done / error
- **Action Bar**: Regenerate, download video
- **Version History**: Supports switching to historical versions

### LeftPanel (Left Technique Library)

- **Category Accordions**: Angle / View / Movement / Style
- **Technique Chips**: Click to apply to selected node
- **Hover Tooltips**: Display technique description and keywords

### RightPanel (Right AI Chat)

- **GlobalConfigBar**: Global model and specification settings
- **ChatMessages**: Chat history
- **StoryboardTabs**: Storyboard result display
  - **OverviewTab**: Overview table
  - **DetailsTab**: Detail cards

## 🔐 Security Features

- **Row Level Security (RLS)**: All tables have row-level security policies enabled
- **User Data Isolation**: Users can only access their own data
- **API Key Hidden**: Sensitive configs exist only on the server side
- **HTTPS Enforced**: All external resource URLs must be HTTPS
